# CHANGELOG | Omni-Matrix Telemetry Engine

### Changed
- Upgraded the execution core from an interpreted Python script to a compiled, memory-safe Rust binary.
- Ported the mathematical padding engine natively into Rust to maintain 100% Institutional Brutalism alignment.
- Implemented fault-tolerant `Result` and `match` extraction to prevent core panics on corrupted ledger rows.


## [1.1.0] - Institutional Reporting Release
### Added
- Mathematical Padding Engine: Enforces strict Brutalist column alignment in raw text outputs.
- Expanded Matrix Logic: Now generates 8 dedicated tables (Time, Global, Metro, Timezone, Content, Device, OS, Architecture).
- Form Factor Parser: Distinguishes between Mobile/Tablet and Desktop/Server hardware.
- Localhost Interception: Correctly silos `file://` and local IP traffic away from live production tracking.
- Semantic Report Headers: Explicitly stamps `Engine Version` directly onto the generated Markdown ledger.

### Changed
- Refactored entire execution architecture from Rust to Fault-Tolerant Python 3 (`omni-matrix-engine.py`).
- Transposed the Time Matrix: Time intervals (Yesterday, 7 Days) are now columns, providing a standardized chronological X-axis for financial review.
- Decoupled Geographic Tuples: Separated Country and Region/State into distinct tracking columns.

### Removed
- Deprecated strict Rust binary constraints to prevent parsing panics on corrupted `.csv` rows.
