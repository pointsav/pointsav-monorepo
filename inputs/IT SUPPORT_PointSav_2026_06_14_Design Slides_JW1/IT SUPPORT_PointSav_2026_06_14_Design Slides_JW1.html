Design System — Totebox OS

Totebox Services  ⇒  Containers

A Totebox is a micro / nano VM that hosts its services as containers — each service plays its own role inside the box.
Totebox
Totebox Services
✲ a micro vm or nano vm
✲A Totebox can run different services, each in its own container.
✲Totebox Services don't have to be transferable — only the Totebox itself does.
Ex Firecracker (AWS) — a micro VM that still lets you run containers inside.
Notes
Build the Totebox like Firecracker — each service in a separate container Use the prototype to create a unikernel as the Totebox Archive Get full services running first

Corporate entities stay separate because they can:
