Notes
Interface OS aggregates across multiple Toteboxes. The interfaces are the priced, proprietary layer; the underlying archives stay customer-owned and portable.
Notes
Which of the six interface services would you build first, and what is its pricing unit — per archive, per query, or per seat? Aggregation runs stateless across Toteboxes — where does that break if a customer holds 200 archives instead of two? What stops a competitor from re-implementing the same interfaces on top of the open base nodes?
