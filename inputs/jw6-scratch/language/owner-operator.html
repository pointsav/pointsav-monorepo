Interface vs Infrastructure

Two roles scale the same fleet for different reasons — the Owner consolidates entities, the Operator runs them — so each reaches the consoles by a distinct, separately-priced path.

✲The Operator reaches F*Keys over PointSav Infrastructure — one entity's archive at a time — and scales by adding infrastructure: on-prem, leased, cloud, or hybrid.
✲Both paths authorize the same way: machine-based, no credential database. The console a person sees is decided by their device's place in the network, not by a stored role — the authenticator-binding problem framed in NIST SP 800-63B-4.
✲Why the split matters commercially: the Owner's interface path is where consolidation is sold; the Operator's infrastructure path is where deployment is sold. Separating them keeps the paid orchestration boundary clean.
