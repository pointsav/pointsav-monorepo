The GIS Engine Stack

The site-selection engine is built on named open-source geospatial components — the concrete answer to what runs beneath the co-location product.
Components
PostGIS — spatial database
stores points, polygons, catchments
QGIS — analysis and authoring
co-location algorithm workbench
OpenStreetMap — base geography
businesses, places, parking polygons
GeoNode — interactive delivery
map layers to the console
Pipeline and economics
✲The flow runs inputs → algorithms → proprietary outputs: public point and polygon data enters, the co-location and catchment algorithms process it, and the ranked co-location databases are the derivative output.
✲Origin-destination catchment data is the one bought input — roughly $50k–$150k per year — joined against free OpenStreetMap geometry on a hexagonal overlay.
✲Every component is open source, so the engine inherits the same portability posture as the rest of the platform — no proprietary geodata runtime to be locked into.
✲For investors the stack maps onto the sovereign-cloud IaaS market Gartner sizes at ~$37B today — the data layer is owned, not rented from a single vendor.
