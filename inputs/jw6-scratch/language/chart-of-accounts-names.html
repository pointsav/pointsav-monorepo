Record Keeping — Bookkeeping

File-Name-as-Reference

The Input Machine derives each file name from the Chart of Accounts, so the name alone places the record — no lookup table required.
A bookkeeping record filed by the Input Machine:
BOOKKEEPINGTITLECO12025_03_07InvoiceProperty ManagementSnow Removal.PDF
01Service — the Totebox service that owns the record (Bookkeeping, Minute Book, IoT).
02Entity — the archive the record belongs to; one Totebox per entity.
03ISO date — sortable, locale-independent (YYYY_MM_DD).
04Document type — Invoice, Receipt, Resolution, drawn from the Chart of Accounts.
05Account path — expense category and description; the same fields a ledger row would carry, now in the name.
06The original PDF / ODT / ODS is the record; the name plus its SHA-256 hash is the only reference anyone needs.
