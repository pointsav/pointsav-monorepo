File System & Partitions

Each archive is a single micro VM partitioned so recovery, the OS, and per-service storage stay isolated — with the best-fit store chosen for each service.

RRecovery — restores the archive to a known-good state; kept apart from operating partitions.
OSOperating system — the Totebox OS itself; read-mostly, separate from records.
S1Service container — one service's records (e.g. Bookkeeping); best-fit database or file system per service.
S2Service container — a second service partition, isolated from the first.
S3Service container — structured services use an embedded DB; document services use the flat-file WORM store.
S4Service container — partition isolation keeps one service's data from reaching another's.
Why partition this way
portable, machine-readable export — per EU Data Act Art. 23–25 (2023)WORM permanence — per ISO 14641:2018no vendor lock-in on read
