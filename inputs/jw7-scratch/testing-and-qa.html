Testing & QA

Every commit to the canonical ledger passes three verification layers before Stage 6 promotion — developer-side testing, integration coverage, and compliance verification that the WORM audit trail is intact.
Developer Layer
1Unit tests — Rust cargo test; every public function has a corresponding test
2Integration tests — end-to-end API calls; real filesystem, no mocks
3Clippy gate — cargo clippy --all-targets -D warnings; zero allowed warnings
Compliance Layer
✲WORM integrity — every WORM record verified append-only before Stage 6 promotion
✲Audit trail — chain of custody confirmed across contributor boundary crossings
✲Stage 6 gate — Command Session promotion only after all checks pass
