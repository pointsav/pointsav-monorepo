PrivateGit OS — Design System

Sets the house style — documentation standards, coding rules, and a shared component library — so a new contributor and an outside auditor read the same platform the same way.
What it entails
✲Adopts IBM Carbon-style thinking for a PointSav Design System hosted on PrivateGit OS — one token set for layout, spacing, grids, and components across Marketing (WordPress), Knowledge (MediaWiki), and Distribution (Fresh RSS) deployments of MediaKit OS.
✲Pins the architecture to four named standards so the same system is legible to an engineer and to a procurement reviewer: C4 Model (context → container → component → code), 4+1 View Model (logical, process, physical, development, plus scenarios), UML 2.5.1 (sequence and component diagrams), and ISO/IEC 42010 (architecture-description conformance, planned).
✲Holds code to the Power of 10 safety-critical rules (NASA/JPL discipline, planned for the Rust codebase): bounded loops, no recursion, explicit return checks — the same constraints that keep a release predictable for a maintainer and verifiable for a risk officer.
