Engineering Posture

Power-of-Ten Safety-Critical Rules

Constrained control flow keeps the codebase statically analysable — the same discipline that makes a kernel reasoned about line by line.
Scope: Rust · WASM · web surfaces. Platform-specific extensions (iOS, Android) are additive — these rules are the primary standard across all environments.
01Restrict control flow to simple constructs — no unbounded recursion or unstructured jumps.
02Give every loop a fixed upper bound so termination can be verified statically.
03Avoid dynamic memory allocation after initialization — predictable footprint, no fragmentation drift.
04Keep functions short and single-purpose so each fits on one reviewable screen.
05Check return values, assert assumptions, and limit scope — per The Power of Ten (Holzmann, NASA/JPL, 2006).
Paired with Rust's ownership model, these rules make the engineering posture legible to developers and the reliability posture legible to investors — and align with the planned seL4 verified-primitives foundation.
Your notes

The Economic Model

Three system-design commitments turn an archive into an asset the customer — not the vendor — controls.
Pillar 1 — Decentralization
A single archive runs on its own with no central coordinator to depend on; a charge applies only when two or more archives are composed together.
Pillar 2 — Auth as Permissions
The pairing itself is the access grant — network topology is the permission system, so there is no credential database to breach or to hold the customer hostage.
Pillar 3 — Portability
Each archive is an open, machine-readable WORM flat-file store — EU Data Act compliance by design, with no egress fee and no proprietary container, so exit is a copy rather than a migration project.
01Decentralized orchestration. A single archive runs free and open source (Totebox OS). Revenue begins only when stateless cross-archive aggregation (Orchestration OS) coordinates two or more archives — the commercial boundary is composition, not the data itself.
02Authorization as permissions. Network topology is the permission system — no central credential database to breach or to hold the customer hostage. Authorization is evaluated per resource at the point of access. — lineage per NIST SP 800-207, Zero Trust Architecture (2020).
03Freely transferable archives. Each archive is a WORM flat-file store in open, machine-readable formats with no egress fee and no proprietary container. Exit is a copy, not a migration project. — aligned with EU Data Act Art. 23–25 (Reg. 2023/2854), egress-fee ban effective Jan 2027.

Ranking and economics
✲A site is scored on an 11-dimension ranking matrix: a primary anchor (Walmart) weighted against secondary anchors (Home Depot, Costco) within ~1.0 km and tertiary places (hospital, university) within ~5.0 km, blended with the primary's sales per square foot. Three-star rows carry both secondaries; single-anchor rows rank lower.
✲The matrix is calibrated to hold the list between 350 and 450 co-locations — the Top 400 markets in North America, the same target in Europe — by tightening or loosening the distance thresholds.
✲Origin-destination catchment data is the one bought input — commercial providers run $50k–$150k per year versus $0 for OpenStreetMap geometry — joined on a hexagonal overlay to read home/work locations of devices entering the parking-lot polygons.
✲Worked example — Franklin, Tennessee (pop. ~85,600, +36.9% growth 2010–2021): two Walmart sites clear the matrix with Home Depot and Lowe's as hardware anchors, Whole Foods and Safeway as food, plus Costco/Sam's, a hospital, and Belmont University in range — a high-tier cluster the engine surfaces, not a hand-picked site.
✲Every component is open source, so the engine inherits the platform's portability posture — the ranked data layer is owned, not rented from one proprietary geodata vendor.

Co-location — Cluster Math

Cluster Methodology

A co-location cluster is defined by degree and distance — the rules that turn a scatter of stores into a ranked site-selection signal.
Degree tiers
01Primary target — the anchor the cluster is built around (e.g. a Walmart Supercentre).
02Secondary / tertiary targets — large-format and civic anchors that reinforce the primary draw.
03First-degree cluster — anchors within 1.0 km of the primary target.
04Up to fifth-degree cluster — the catchment expanded out to a 5.0 km threshold.
What the rules produce
✲The two thresholds — 1.0 km first-degree, 5.0 km fifth-degree — are the whole sorting rule. Every candidate site is scored by how many target tiers fall inside each ring.
✲Tiles for each target type — Walmart, Home Depot, Costco, hospital, higher-education — feed an 11-row ranking matrix with primary / secondary / tertiary weighting.
✲The output is a cluster record with a sales-per-square-foot proxy, ready to join against catchment and spend layers in the GIS engine.
✲Because the rule is purely geometric, it is reproducible and auditable — no opaque model weights stand between the input geography and the ranked output.

Why this substrate

✲For developers this is a credible, inspectable rendering path; for the F12 checkpoint it means the human-authorization HUD looks the same on every deployment.
