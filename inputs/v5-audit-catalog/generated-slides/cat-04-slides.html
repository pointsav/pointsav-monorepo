Totebox Orchestration

The Economic Model

Three system-design commitments turn an archive into an asset the customer — not the vendor — controls.
01Decentralized orchestration. A single archive runs free and open source (Totebox OS). Revenue begins only when stateless cross-archive aggregation (Orchestration OS) coordinates two or more archives — the commercial boundary is composition, not the data itself.
02Authorization as permissions. Network topology is the permission system — no central credential database to breach or to hold the customer hostage. Authorization is evaluated per resource at the point of access. — lineage per NIST SP 800-207, Zero Trust Architecture (2020).
03Freely transferable archives. Each archive is a WORM flat-file store in open, machine-readable formats with no egress fee and no proprietary container. Exit is a copy, not a migration project. — aligned with EU Data Act Art. 23–25 (Reg. 2023/2854), egress-fee ban effective Jan 2027.

Totebox OS — Communications

Communications as Flat Files

Email, chat, calendar, and conferencing are stored one file per message — not locked inside a mail server's database.
The record model
One file per message; one Totebox per person
emailchatcalendarconferencing
Each message lands write-once. Deletion is a tombstone, never an erase — the archive stays append-only and auditable.
Why Maildir
✲Lock-free tmp/new/cur directory layout — safe to read and write over a networked file system without a database daemon. — per Maildir (D. J. Bernstein).
✲A proven enterprise format, not a bespoke schema — documented as a first-class IMAP mailbox store. — per Dovecot Maildir documentation.
✲Maps directly onto the WORM, non-rewriteable record standard regulated firms already recognize. — per SEC Rule 17a-4 (2022).

Standards Precedent

The Portability Precedent

User-owned data stores with app/data separation are an established standard — the freely transferable archive is the same idea, applied to the business entity.
W3C Solid pods
✲A user owns the pod; applications connect to it rather than absorbing the data. — per Solid Protocol (W3C Solid Community Group, 2022).
✲Providers are interchangeable — the pod migrates between hosts without rewriting the application.
Totebox archive
App/data separation at the entity level
archive = the podconsole = the appWORM flat-file
The archive is application-independent: any conforming console reads it, and the customer keeps the canonical copy. — portability framing per GDPR Art. 20, "structured, commonly used, machine-readable."

Search the Archive

A structured query surface reads the flat-file archive directly — filter, locate, and export without a separate index service.
Query surface
✲Filter fields map to the file-name convention — date range, document type, counterparty, account.
✲Worked example: a Federal Register query resolves to the underlying source files, each held write-once in the archive.
✲Results export in open, machine-readable form — CSV / JSON — so the output travels with the customer.
Why it stays portable
The file name is the reference
no proprietary indexhuman-readable pathsCSV / JSON export

Catchment Data Economics

Origin-destination catchment data is expensive to license — sourcing it once and delivering it as interactive layers is the integration value.
Sourcing
✲Commercial origin-destination catchment feeds run roughly $50k–$150k per year — a recurring cost most SMBs cannot carry alone.
✲The platform sources the feed once and exposes it through the orchestration tier, spreading the cost across archives.
Delivery
Interactive layers, not raw dumps
GeoNode deliverytiled layersper-archive access
Catchment results are delivered as interactive map layers the customer queries directly — and the derived outputs export with the archive, not the upstream license.

MediaKit & PrivateGit OS

One Navigation, Every Deployment

The same documentation flow appears whether content ships through MediaKit OS or PrivateGit OS — consistency is what makes the platform portable to operate.
01Landing → Newsroom. A reader moves from a landing page into the newsroom along the same path on every deployment surface.
02Marketing → Distribution. The marketing-to-distribution flow is identical across MediaKit OS (WordPress / MediaWiki / Fresh RSS) and PrivateGit OS.
03Docs as code. Documentation lives beside the source it describes, so it travels with the archive and survives a switch of host. — per Write the Docs, Docs as Code.
