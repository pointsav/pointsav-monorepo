Engineering Posture

Power-of-Ten Safety-Critical Rules

Constrained control flow keeps the codebase analysable — the same discipline that lets a kernel be reasoned about.
01Restrict control flow to simple constructs — no unbounded recursion or unstructured jumps.
02Give every loop a fixed upper bound so termination can be verified statically.
03Avoid dynamic memory allocation after initialization — predictable footprint, no fragmentation drift.
04Keep functions short and single-purpose so each fits on one reviewable screen.
05Check return values, assert assumptions, and limit scope — per The Power of Ten (Holzmann, NASA/JPL, 2006).
Paired with Rust's ownership model, these rules make the engineering posture legible to developers and the reliability posture legible to investors — and align with the planned seL4 verified-primitives foundation.
