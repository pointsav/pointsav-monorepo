# News Release Typography and Layout Standards

This document defines the strict ISO 24495-1 Plain Language formatting rules for all corporate news syndication across the organization.

## 1. Universal Formatting Rules
* **Alignment:** All text must be strictly left-aligned (ragged right). Justified text is prohibited to maintain accessible visual tracking.
* **Capitalization:** Use Title Case for headlines and corporate entities. All-caps blocks (e.g., WOODFINE, VANCOUVER) are prohibited.
* **Geographic Expansion:** Spell out provinces and states completely (e.g., British Columbia, Columbia Británica). Two-letter abbreviations (e.g., BC) are prohibited to prevent international jurisdictional collisions.

## 2. Printed News Release Structure
* **Header:** Bold, Title Case.
* **Dateline Separator:** Use an en-dash (–) or em-dash (—) to separate the dateline from the opening narrative paragraph.
* **Format:**
  **For Immediate Release**
  **[Headline Title Case]**
  **Vancouver, British Columbia – [Month] [Day], [Year] –** [Narrative begins...]

## 3. Web News Release Structure
* **Header Tags:** Use H2 for the release category and H1 for the headline.
* **Metadata Separator:** Use a pipe symbol (|) to separate dateline elements on a dedicated line.
* **Format:**
  **[Corporate Entity] — [Release Category]**
  **[Headline Title Case]**
  **For immediate release | Vancouver, British Columbia | [Date]**
  [Narrative begins...]
