## Review expectations

Maintainers commit to:

- Acknowledging opened issues and PRs within five business days, even if
  substantive review takes longer.
- Reviewing their assigned `CODEOWNERS` PRs within two weeks of opening.
- Participating in ADR discussions within the 72-hour comment window per
  [GOVERNANCE.md §2.2](./GOVERNANCE.md).
- Responding to `[security]` emails within 72 hours per
  [SECURITY.md](./SECURITY.md).

If a maintainer is unavailable for an extended period (vacation, illness,
sabbatical), they should note it on the relevant GitHub issue or in a
commit to this file so contributors know not to block on them.
