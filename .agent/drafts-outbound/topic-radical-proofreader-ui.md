# TOPIC: Radical Proofreader UI & Rust Distillation

This topic covers the architectural decisions behind the radical UI redesign of the Proofreader console and the implementation of a pure-Rust training distillation loop.

## The Radical UI Redesign
The `app-console-proofreader` has transitioned from a strictly "plaintext-clean" server-rendered HTML interface to an award-winning, premium web experience. While maintaining the core constraints (no JavaScript frameworks, server-side rendering, and the strict "no-train-on-tenant-text" disclosure), the new UI implements a two-tier design token architecture.

Tokens are strictly separated:
- **Generic Tokens**: Baseline CSS variables and layout primitives routed from `/pointsav-design-system`.
- **Tenant Overrides**: Specific branding, typography, and logos routed from `/woodfine-media-assets`.

This allows the UI to adapt dynamically to the tenant's brand identity without tightly coupling the console's source code to a specific customer.

## Pure-Rust Training Distillation
To close the machine learning loop, the system extracts the `apprenticeship/prose-edit` JSONL corpus generated by the proofreader and distills it into a `training_dataset.jsonl` format expected by `service-slm`.

We selected a pure-Rust approach (via `tool-proofreader-trainer` or an extended `service-proofreader` subcommand) over Python scripts. This unifies the operational tooling ecosystem, guarantees type safety when parsing the complex event-pair schema (draft-created, draft-refined, creative-edited), and integrates seamlessly into the cargo workspace. The distillation isolates the operator's accepted and edited verdicts, providing perfect instruction-output pairs for the Stage-2 DPO (Direct Preference Optimization) pipeline.
