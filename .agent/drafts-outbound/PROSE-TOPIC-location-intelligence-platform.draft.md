# Location Intelligence Platform

A co-location cluster analysis platform for retail site selection and office development
planning. Designed as a sovereign, flat-file, open-standards alternative to commercial
Location Intelligence SaaS products.

## What it does

The platform ingests retail store locations from OpenStreetMap and the Overture Maps
Foundation, computes five-degree co-location clusters around Primary Target retailers
(Walmart Supercentres in North America; IKEA in Europe), and renders the results as
an interactive layered map at gis.woodfinegroup.com.

The output answers one question: *which retail clusters support adjacent office
development?*

## Five-degree co-location algorithm

A co-location cluster is scored by how many complementary retailers are present within
defined radii of a Primary Target location.

**Primary Target** — Walmart Supercentre (North America) or IKEA (Spain, Italy, Greece,
Nordics, Poland). The anchor that draws the highest sustained foot traffic.

**Secondary anchors** — Home Depot, Costco, Lowe's (North America); Leroy Merlin,
Brico Depôt, Carrefour (Europe). Must be within 1.0 km of the Primary Target.

**Tertiary anchors** — Hospitals, medical centres, universities. Must be within
5.0 km of the cluster centroid.

| Degree | Composition |
|---|---|
| 1st | Primary Target alone |
| 2nd | Primary + one Secondary within 1.0 km |
| 3rd | Primary + two or more Secondaries within 1.0 km |
| 4th | Primary + Secondary(ies) + one Tertiary within 5.0 km |
| 5th | Primary + Secondary(ies) + two or more Tertiaries within 5.0 km |

**Calibration**: if 5th-degree clusters exceed 10% of all Primary Target locations
in a given country, the Secondary radius tightens from 1.0 km to 3.0 km, then to
0.8 km if over-represented at the tighter threshold. This prevents grade inflation
in dense urban markets.

**Europe variant**: IKEA serves as the Primary Target across all European markets in
scope. Secondary anchors adapt to local equivalents (Leroy Merlin, Brico Depôt).
Tertiary logic is identical.

## Three-layer map architecture

**Layer 1 — Base locations**: All 27,444+ individual store coordinates, sourced from
OpenStreetMap via the Overpass API. Coloured by brand family (Department / Hardware /
Food / Furniture). Available as a toggle; not the default view.

**Layer 2 — Co-location clusters**: The primary working view. Each cluster marker
encodes degree by size and colour saturation — 5th-degree clusters are visually
dominant. Clicking a cluster opens an index card with city, national rank, secondary
anchors present, and service-places within 5 km.

**Layer 3 — Catchment radius**: A circular buffer (default 75 km; 25–30 km in dense
urban markets; up to 150 km in sparse/rural markets) around each cluster centroid.
This boundary defines the scope for origin-destination mobility data procurement:
only device pings from origins within the radius are relevant to the cluster's
trade-area analysis.

## Data sovereignty

All data, tiles, and algorithms are customer-owned:

- Retail location records: JSONL files in SafeGraph-compatible schema, stored in
  `cluster-totebox-personnel-1/data/service-business/`
- Service-places records: Overture Maps Foundation data, stored in
  `cluster-totebox-personnel-1/data/service-places/`
- Map tiles: PMTiles format, generated by Tippecanoe, served by nginx — no CDN
  dependency required at runtime
- Co-location algorithm: open Python script (`build-clusters.py`) that reads from
  the Totebox and writes cluster records

If the GIS gateway is lost, all data persists in the Totebox Archive and can be
served to a replacement gateway instance.
