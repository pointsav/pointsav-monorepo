# readme-workspace

The README at a workspace root (e.g., `~/Foundry/`). It identifies the
workspace, says which deployment it represents, and points cold readers
at the constitutional charter (`DOCTRINE.md`), the operational guide
(`CLAUDE.md`), the open-items queue (`NEXT.md`), and the deployment
passport (`MANIFEST.md`).

Workspace READMEs are the most public-facing of the three README
sub-types. Bilingual pair required.
